# World History Timeline — build plan

Interactive static site. No build step, no dependencies. Open `index.html` in a browser.

## Status

- [x] Phase 1 — core app (done, built inline): `index.html` (log-scale timeline, tooltips, drawer, table view), `data.js` (~80 civilizations), `articles/article.css`, reference article `articles/ancient-egypt.html`.
- [ ] Phase 2 — articles (parallel): one HTML article per entry in `data.js`, generated by `build-articles.sh`. Each generation task is independent → embarrassingly parallel, fresh context per agent.
- [ ] Phase 3 — optional review: `./build-articles.sh` again picks up failures (it skips existing files); spot-check a few articles for factual sanity.

## Article spec (agents: this is your full instruction — you have no other context)

You are given ONE line from `data.js` describing a civilization:
`{id, name, continent, start, end, blurb, wiki}` — years negative = BCE, positive = CE, 2026 = present.

Write exactly one file: `articles/<id>.html`. Touch nothing else.

Requirements:
1. Copy the exact HTML structure of `articles/ancient-egypt.html` (read it first). Same
   `<head>` (stylesheet `article.css`), same classes: `.back`, `h1`, `.meta`,
   `article` with 3–4 paragraphs, `.contemporaries` box, `.more` Wikipedia link.
2. `<title>` = `<name> — World History`. `.meta` = `<Continent name> · <start> – <end>`
   formatted like "3100 BCE – 30 BCE", "618 CE – 907 CE", or "1368 – 1644"; use
   "present" for 2026. Continent display names: africa=Africa, europe=Europe, asia=Asia,
   namerica=North America, samerica=South America, oceania=Oceania.
3. Article body: 3–4 paragraphs, ~250–400 words total, factual, encyclopedic but readable.
   Cover origin, peak/defining achievements, decline or legacy. No invented facts, no
   citations needed; stay with mainstream consensus history.
4. `.contemporaries` box: 3 bullets, "Meanwhile, elsewhere" — what was happening on
   OTHER continents during this civilization's timespan. Must be historically accurate.
5. `.more` link: use the `wiki` URL from the data line, with `target="_blank" rel="noopener"`.
6. Plain HTML, no scripts, no external resources besides `article.css`.

## Parallelization model

`build-articles.sh` consumes this plan: it greps civilization ids from `data.js`
(one entry per line — keep that format) and spawns one fresh-context
`claude -p` agent per article, `CONCURRENCY` at a time. Agents share nothing;
each reads only this spec, the reference article, and its own data line.
Idempotent: existing non-empty articles are skipped, so re-running retries only failures.
